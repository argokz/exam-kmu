# exam-kmu

Контрольная по Python: генерация задания и проверка через GPT с сохранением результатов в PostgreSQL.

## Как открыть в Google Colab

1. Откройте в браузере:
   ```
   https://colab.research.google.com/github/argokz/exam-kmu/blob/main/Exam_KMU.ipynb
   ```
2. Либо: **File → Open notebook → вкладка GitHub** → введите `argokz/exam-kmu` → выберите `Exam_KMU.ipynb`.

## Настройка (один раз, делает преподаватель)

Ключи **автоматически считываются с GitHub**. Варианты:

**Вариант A — секретный Gist (рекомендуется)**  
1. Создайте на GitHub [новый Gist](https://gist.github.com/) (галочка «Secret»).  
2. Имя файла: `secrets.json`. Содержимое:
   ```json
   {"OPENAI_API_KEY": "sk-...", "DB_PASSWORD": "пароль_БД"}
   ```
3. Сохраните, откройте «Raw» и скопируйте URL.  
4. В ноутбуке в первой ячейке вставьте: `GITHUB_SECRETS_URL = "этот_raw_url"`. Сохраните.  
Студенты не видят ключи — ноутбук подгружает их при запуске.

**Вариант B — файл в этом репозитории (приватном)**  
1. В репо создайте файл `secrets.json` (см. `secrets.example.json`), добавьте его в `.gitignore` и **не коммитьте** реальный файл с ключами — либо храните ключи только в приватном репо и дайте доступ по токену.  
2. Либо: создайте файл в репо, получите raw-URL (например `https://raw.githubusercontent.com/argokz/exam-kmu/main/secrets.json`). Для приватного репо в ноутбуке укажите также `GITHUB_TOKEN = "ваш_токен"` (Scope: `repo`).  
3. В ноутбуке: `GITHUB_SECRETS_URL = "raw-URL"`.  

После настройки студентам вводить ничего не нужно — они открывают ноутбук и работают.

## База данных

Сервер: `145.249.247.138:5440`, БД: `exam_kmu`, пользователь: `postgres` (или свой).

1. Создайте базу (если ещё нет):
   ```sql
   CREATE DATABASE exam_kmu ENCODING 'UTF8';
   ```
2. Создайте таблицу (выполните `schema.sql` в `psql` или запустите `init_db.py` с заданным `DB_PASSWORD`).

Результаты сохраняются в таблицу `results` (ФИО, группа, кредит, вариант, код, оценка 0–100, комментарий, дата).

## Порядок для студента

1. Указать **ФИО**, **группу**, **кредит**.
2. Нажать **Сгенерировать задание** — появится одно задание (вариант определяется по ФИО/группе/кредиту).
3. Написать решение в ячейке с `student_code` и запустить ячейку.
4. Нажать **Проверить** — отобразится оценка и комментарий; результат записывается в БД.
